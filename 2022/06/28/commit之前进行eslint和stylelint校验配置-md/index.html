    • commit之前进行eslint和stylelint校验配置.md

    2022-06-28- 2022-06-29
    前端-git

    本次文章只针对，项目中已经有了eslint的配置和stylelint的配置，如果没有配置，请先配置eslint和stylelint。如不了解git hooks可以了解一下

    想要的效果

    • 在commit的时候提前进行eslint，stylelint的校验，如果校验不通过的话，进行报错提示。

    • 对老代码无影响，只针对变化的部分。

    使用的工具

    • husky

    husky 继承了Git下的所有钩子，在触发钩子的时候，husky可以阻止不合法的commit，push等。 注意在使用之前，必须要现将代码放在git 仓库中，否则本地没有.git文件，就没有地方去继承钩子了。

    在这里我们主要使用的是pre-commit在提交之前做一些东西，如果想要了解更多可以查看官网

    • lint-staged

    简单来说就是，在你提交的文件中，执行一下自定义的命令。了解更多查看这篇文章。

    实操

    安装依赖

      npm install husky --save-dev // 安装husky
  npm install lint-staged --save-dev // 安装 lint-staged

    联系git hook

      npx husky install

    创建一个hook

      npx husky add .husky/pre-commit "npm run lint-staged" // 引号部分是要执行的lint-staged命令

    配置package.json

      {
    "scripts": {
      "lint-staged": "lint-staged"
      "prepare": "husky install"
    },
    "husky": {
      "hooks": {
        "pre-commit": "lint-staged"
      }
    },
    "lint-staged": {
      "* */*.{vue,js}": "eslint", // eslint 校验
      " **/ *.{vue,htm,html,css,less}": "stylelint"// stylelint校验
    }
  }

    遇到的问题

    • 如果commit的时候报一个文件找不多的错，可能是node 的版本太低

    • 可以先尝试写一个简单的例子，尝试一下husky有没有执行。
